# Файл скрипта (клиент): порядок строк и полей

Команда консоли **`execute_script путь`** открывает файл на **машине клиента** (путь относительно каталога текущего скрипта или абсолютный). Пустые строки и строки с `#` в начале пропускаются.

Пока идёт выполнение файла, ввод для многострочных команд (`insert`, `update`, …) читается **из того же файла** (подмена `InputHandler` на `Scanner` по файлу — см. `client.command.ExecuteScriptRunner`).

По сети на сервер уходит:

1. **`ScriptSessionBeginRequest`** с токеном = нормализованный абсолютный путь к этому файлу (сервер кладёт токен в стек и проверяет рекурсию).
2. Для **каждой** непустой команды в файле — обычный запрос (`ShowRequest`, `InsertRequest`, …), как если бы строку ввели вручную в консоли.
3. **`ScriptSessionEndRequest`** с тем же токеном (сервер снимает вершину стека).

Вложенный **`execute_script вложенный.txt`**: относительный путь разрешается от каталога **текущего** файла; для вложенного файла снова выполняются шаги 1–3.

---

## `insert <ключ>`

После строки `insert <ключ>` идут поля **нового** `MusicBand` (как `MusicBandReader.read`):

1. Название группы (не пустое)
2. Координата X (вещественное, запятая или точка)
3. Координата Y (≤ 968)
4. Количество участников: пустая строка = `null`, иначе целое > 0
5. Количество альбомов (> 0)
6. Жанр: пусто = нет жанра; иначе имя enum (`POST_ROCK`) или номер из подсказки жанров (1-based)
7. Имя фронтмена: пусто = нет персоны; иначе далее три строки:
   - Дата рождения `yyyy-MM-dd HH:mm:ss` или пусто = нет даты
   - Рост: пусто = нет роста; иначе целое > 0
   - ID паспорта (обязательно, не длиннее 28 символов)

---

## `remove_greater`

После строки `remove_greater` (без аргументов) — поля эталонного `MusicBand` в том же формате, что и для `insert` (без ключа в отдельной строке).

---

## `replace_if_greater <ключ>` / `replace_if_lower <ключ>`

Как при интерактивном вводе: сначала ключ, затем поля новой группы; имя сравнивается с текущим именем элемента с этим ключом (строго больше / строго меньше).

---

## `update <id>`

Поля построчно в порядке **`MusicBandReader.readForUpdate`** (пустая строка = оставить прежнее, где допустимо; для `null` — как в консоли, со строкой подтверждения `yes`/`no`).

---

## Клиентская команда `save`

В файле скрипта не используется (нет в протоколе). Закомментируйте строку или удалите.
